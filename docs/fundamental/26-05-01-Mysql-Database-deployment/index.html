Mysql Database deployment

Why we need many ways to deploy ?​

  • HA: zero downtime.
  • Scalability: read, write
  • Consistency
  • Performance
  • Cost

Replication​

Types of replication synchronization:

  • asynchronous (default): one-way, in that 1 server acts as a source, 1 or more other servers act as replicas.

Replication methods:

  • binlog: https://dev.mysql.com/doc/refman/9.7/en/replication-configuration.html

Single instance​

App ---> mysql (1 node only, nothing to say)

traffic layer → proxy/load balancer (ProxySQL, HAProxy, or MySQL Router) control layer → failover manager (Orchestrator, MHA, or even manual for testing) data layer → MySQL nodes (1 primary + replicas)

  • 1 primary (write)
  • 2 replicas (read)
  • GTID enabled (so failover is easier)
    • GTID stand for global transaction identifier. It's unique identifier created and assigned to every tx that is commited in primary server.
